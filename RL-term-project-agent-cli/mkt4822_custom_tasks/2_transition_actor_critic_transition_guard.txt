Task: Build an Actor-Critic RL application with safe transition handling.

Environment:

* 1D grid world with states 0..9.
* Start state is 0.
* Goal state is 9.
* Actions are 0 = left and 1 = right.
* If movement would go outside the grid, clamp the resulting state.
* The transition helper must never return a state outside [0, 9].
* Reward is +10 at goal and -1 otherwise.
* Episode ends at goal or max_steps=35.

Actor-Critic:

* Use tabular actor preferences.
* Use tabular critic values.
* Use softmax during training.
* Use greedy evaluation.
* Use TD error for actor and critic updates.

Requirements:

* Train for 100 episodes.
* Evaluate for 10 episodes.
* Print reward, steps, and termination reason for each episode.
* Generate generation_report.csv with episode, total_reward, steps, termination.
* Print a textual policy visualization.
* Include random.seed(42).
* Include type hints.
* Include if **name** == "**main**".
* Standard library only.
* Every loop must have a max_steps guard.
