Task: Build a runnable Actor-Critic RL application that clearly distinguishes successful termination from timeout.

Environment:

* 1D grid world with states 0..9.
* Start state is 0.
* Goal state is 9.
* Actions: 0 = left, 1 = right.
* Clamp boundaries.
* Reward is +10 at the goal and -1 otherwise.
* Episode ends when the goal is reached or max_steps=30 is exceeded.

Termination requirements:

* If the goal is reached, report termination reason as "goal".
* If max_steps is reached before the goal, report termination reason as "timeout".
* Every episode must terminate safely.

Actor-Critic:

* Use tabular actor preferences.
* Use tabular critic values.
* Use softmax training policy.
* Use greedy evaluation policy.
* Use TD error.

Outputs:

* Train for 80 episodes.
* Evaluate for 10 episodes.
* Print reward, steps, and termination reason for every episode.
* Generate generation_report.csv with columns episode, total_reward, steps, termination.
* Include textual policy visualization.
* Include random.seed(42).
* Include if **name** == "**main**".
* Use type hints.
* Standard library only.
